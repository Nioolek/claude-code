## 1. 核心贡献

ClawBench 是一个**面向真实世界日常在线任务**的 AI Agent 评估基准，核心创新点：

| 维度 | ClawBench | 现有基准 (OSWorld/WebArena) |
|------|-----------|---------------------------|
| **运行环境** | 真实生产网站 | 静态 HTML 沙箱 |
| **任务数量** | 153 个任务 | 通常 <100 |
| **覆盖平台** | 144 个真实网站 | 有限模拟网站 |
| **任务类别** | 15 个生活类别 | 有限场景 |
| **动态性** | 完整动态网页、JS 渲染 | 静态页面 |
| **真实性** | Cookie 同意、反机器人防御 | 简化环境 |

## 3. 技术架构

### 3.1 安全评估机制

ClawBench 直接在**真实生产网站**上运行，但通过轻量级拦截层确保安全：

```
┌─────────────────────────────────────────────────────────────┐
│                  ClawBench 安全拦截架构                      │
│                                                             │
│   Agent → Chrome Extension → CDP Server → 真实网站          │
│                              │                              │
│                              ▼                              │
│                    ┌─────────────────┐                      │
│                    │  拦截层 (Intercept) │                   │
│                    │                 │                      │
│                    │ • 监控所有请求   │                      │
│                    │ • 识别提交请求   │                      │
│                    │ • 拦截最终提交   │                      │
│                    │ • 返回模拟成功   │                      │
│                    └─────────────────┘                      │
│                                                             │
│   效果：Agent 体验完整流程，但不会产生真实交易                │
└─────────────────────────────────────────────────────────────┘
```

**关键设计**：
- 仅拦截**最终提交请求**（如订单确认、表单提交）
- 之前的所有浏览、填写、验证操作都在真实网站上执行
- 拦截后返回模拟成功响应，Agent 认为任务已完成
- 防止真实扣款、真实提交申请等副作用

### 3.2 五层记录基础设施

ClawBench 同步捕获多模态数据，提供全面的行为分析：

```
┌─────────────────────────────────────────────────────────────┐
│              ClawBench 五层数据记录系统                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Layer 1: 会话视频重放 (Session Video Replay)                │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 完整任务执行过程的屏幕录制，支持时间轴回放             │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  Layer 2: 每步动作截图 (Step Screenshots)                    │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 每个动作前后的页面截图，用于视觉状态追踪               │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  Layer 3: HTTP 请求流量 (HTTP Request Traffic)               │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 完整的网络请求/响应日志，包括 headers、cookies、body   │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  Layer 4: 代理推理日志 (Agent Reasoning Logs)                │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ LLM 的思考过程、决策依据、工具调用参数                  │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  Layer 5: 低级浏览器操作 (Low-level Browser Operations)      │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ Playwright/Selenium 级别的原始操作记录                 │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**数据用途**：
- 错误诊断：定位失败原因（视觉理解？表单填写？导航错误？）
- 模型对比：不同模型的行为模式差异
- 研究分析：Agent 行为的定量和定性研究

## 4. 评估方法

### 4.1 代理评估器 (Agent Evaluator)

ClawBench 使用**Claude Code 子代理**作为评估器，基于人类参考轨迹进行步骤级对齐：

```
┌─────────────────────────────────────────────────────────────┐
│                  ClawBench 评估流程                          │
│                                                             │
│  1. 人类专家执行任务 → 生成参考轨迹 (Reference Trajectory)   │
│     • 记录每一步操作                                        │
│     • 标注关键字段和决策点                                  │
│                                                             │
│  2. Agent 执行任务 → 生成执行轨迹 (Execution Trajectory)     │
│     • 记录所有动作和中间状态                                │
│     • 捕获失败点和错误信息                                  │
│                                                             │
│  3. Claude Code 子代理对比 → 生成评估报告                    │
│     • 步骤级对齐检查                                        │
│     • 识别偏离和错误                                        │
│     • 生成结构化失败诊断                                    │
│                                                             │
│  4. 输出二元结果 + 诊断报告                                  │
│     • Success/Failure                                       │
│     • 失败原因分类                                          │
│     • 改进建议                                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 评估指标

| 指标 | 说明 |
|------|------|
| **任务成功率** | 完成任务的比例（二元：成功/失败） |
| **步骤完成率** | 正确执行的步骤比例 |
| **失败原因分类** | 导航错误、表单填写错误、验证失败等 |
| **平均步骤数** | 完成任务所需的平均操作数 |
| **平均时间** | 完成任务的平均耗时 |

### 5.3 失败原因分析

| 失败类型 | 占比 | 说明 |
|----------|------|------|
| **表单填写错误** | ~35% | 字段格式错误、必填项遗漏、信息提取错误 |
| **导航错误** | ~25% | 找不到正确入口、页面跳转错误、死循环 |
| **验证失败** | ~20% | 无法通过验证码、登录失败、会话过期 |
| **视觉理解错误** | ~12% | 误认按钮、遗漏关键信息、布局变化 |
| **其他** | ~8% | 超时、网络错误、网站结构变化 |

## 10. 总结

**ClawBench 的核心价值**：

> 将 AI Agent 评估从"玩具基准"推向"真实世界挑战"，揭示了当前模型在日常任务上的真实能力边界。

**关键发现**：
- 最佳模型 (Claude Sonnet 4.6) 仅能完成 33.3% 的任务
- 真实世界任务难度远高于静态基准（65-75% → 33.3%）
- 写操作和跨平台导航是主要瓶颈

**研究贡献**：
- 153 个真实日常任务，覆盖 144 个网站
- 安全评估机制（拦截层）
- 五层记录基础设施
- 代理评估器

**开源地址**：论文和代码已开源，支持社区复现和扩展
